Inleiding

Waarom?

Inleiding

Ik heb gekozen voor dit bedrijf omdat zij een front-end developer nodig hadden op CMD Stagebank. Als CMD’er wil ik graag meer leren over front-end. Ik wil zelf werken als front-end developer. Ik dacht: als zij een front-end developer zoeken dan is dit voor mij misschien wel de beste plek om meer te leren over front-end zodat ik mezelf kan ontwikkelen als junior front-end developer.

Om deze stageplek te vinden heb ik gekeken op CMD stagebank. Ik ging op zoek naar een front-end developer stage. Ik bekeek de eisen van het stagebedrijf en maakte daar een verhaaltje van in mijn motivatiebrief. Ik verstuurde een mail naar het bedrijf en kreeg al snel antwoord. Om een stage te vinden die begon in februari 2022, heb ik de sollicitatie mail gestuurd in begin oktober 2021.

De stage is voor mij goed bevallen. Ik heb veel geleerd. Ik heb mijzelf kunnen verbeteren als front-end developer. Om te werken in dit bedrijf had ik iets nieuws moeten leren. Zo maak je een omweg naar je doel toe. Meestal is een omweg iets negatiefs maar voor dit bedrijf vond ik het iets positiefs. Want zo heb ik weer nieuwe dingen geleerd wat ik later ook misschien wel kan gebruiken.

In dit stageverslag kunt u zien wat ik allemaal heb gedaan binnen CrossmarX. Van Onderzoeken naar het werkend maken van een dashboard. Wat ik heb gedaan met welke leerdoelen. U kunt ook kennismaken met de 2 geïnterviewde om meer te weten komen over CrossmarX. De observatieopdrachten zijn gemaakt met een interview om niet alleen maar hoeven te lezen. Met dit geprogrammeerde stageverslag neem ik u mee op een interactievere manier door mijn stage.

Met mijn stage heb ik het meest gewerkt aan de “Feel good”- aka Am chart opdracht. Dat was ook mijn hoofddoel met mijn stage. Met deze opdracht heb ik geleerd om te werken met JavaScript en Velocity.

Bedrijf

Waar?

Bedrijf

CrossmarX is een softwarebedrijf dat in eigen beheer een “low code” product ontwikkelt.
Zo maken zij platforms voor bedrijven. Voor ZorgMinder hebben zij een applicatie ontwikkeld die voor de medewerkers, backoffice maar ook voor cliënten het centrale en communicatieve platform is. Voor Gezondheid[e]Stichting hebben zij een platform gemaakt waarmee Health[e]Foundation informatie kan verstrekken aan gezondheidsmedewerkers over de hele wereld.

CrossmarX ondersteunt hun klanten niet uitgebreid met design. Wel geven zij bij sommige momenten vorm aan hun werk. Bijvoorbeeld de “feel-good” -omgeving die ik ga vormgeven. Design is een klein deel wat zij doen als bedrijf.
Alle medewerkers zitten in 1 grote ruimte. Verder zijn er nog kleine ruimtes om te werken. Ik zal in de eerste week veel samenwerken met Tim Houtenbos om in het bedrijfsleven te komen van CrossmarX. Verder zal ik samen met Matthijs Lambooij samenwerken met de “feel-good” -omgeving.

Bij dit bedrijf ging ik aan de gang met het realiseren van de “Feel good”-omgeving. Dat was mijn hoofddoel. Bij de “Feel-good”-omgeving opdracht komt veel JavaScript bij kijken. Dat was ook wat ik graag wilde leren met mijn stage. Verder heb ik veel kunnen leren over front-end wat mij zal gaan helpen om een baan te vinden als front-end developer.

Wat heb ik geleerd?

Leerdoelen

Leerdoel 1: Begrijpen en kaderen – Deskresearch

Aan het begin van mijn stage heb ik kunnen werken aan dit leerdoel. Bij elke opdracht die ik heb gekregen heb ik deskresearch gedaan om meer het probleem te begrijpen van de opdracht. Zo kon ik het probleem beter oplossen. Deskresearch is gedaan op verschillende maten. De één uitgebreider onderzoek dan de ander. Onderzoek documentatie is te vinden bij extra.

Waar ik het meest vooronderzoek heb gedaan is bij de “Feel good”-omgeving (aka Am charts). Daar heb ik tutorials gevolgd om de structuur van Am chart te begrijpen en hoe verschillend elke chart is om te maken. Zie hier gemaakt werk.

Dit leerdoel heeft mij niet geleerd hoe je beter moet onderzoeken maar hoe diep je kan onderzoeken. Door het onderzoeken heb ik nu meer kennis opgedaan wat mij hielp op stage maar wat mij ook gaat helpen in mijn carrière.

Op dit moment heb ik nog niet zoveel gedaan met dit leerdoel. In tegendeel vindt Tim dat ik wel wat heb gedaan met dit leerdoel. Als ik iets gemaakt heb en het is goed waar geen uitleg bij nodig is, is het voor hem gewoon goed. Tim vertelt mij welke methode of tool moeten gebruiken om een probleem op te lossen. Om toch zo’n beetje al mijn leerdoelen te halen, heb ik meer gedaan bij Leerdoel 6. In mijn Stageplan staat er dat ik elke 1 of 2 weken ga werken met JavaScript om mijn JavaScript te verbeteren. Maar ik heb het elke dag gegaan waardoor ik veel heb kunnen leren met Leerdoel 6. Ook vind ik van mezelf dat ik meer heb gedaan bij Leerdoel 7 vanwege mijn zelf bedachte opdracht: Redesign CrossmarX. Hier heb ik veel geleerd over Sass.

Leerdoel 5: Ontwikkelen en reflecteren – Kritisch

Hierbij heb ik niet elke 2 weken terug gekeken naar mijn werk om te verbeteren omdat ik voor een lange tijd bezig was met 1 grote opdracht: Am chart. (Feel good-omgeving). Wel heb ik deze leerdoel uit kunnen voeren waarbij opdrachten zijn verbeterd. Met deze leerdoel heb ik mijn werk steeds kunnen verbeteren waarbij ik in zag dat niet alles in 1 x perfect kan zijn.

De opdrachten waar uitgebreide veranderingen voor zijn gemaakt zijn:

  • ColorIsDark
  • Strandeiland JS
  • Am Chart

Verbeteringen staan onderaan op de pagina van de single portfolio pagina’s.

Bij de Am chart opdracht is de hele tijd veranderingen gemaakt. Eerst was het zo dat het dashboard zou worden gebruikt voor 1 doel waardoor we hard code konden gebruiken om de opdracht uit te voeren. Maar dat veranderde. Dat werd meer statisch zodat we het op meerdere plekken kunnen gaan gebruiken. Wanneer ik een begin had gemaakt met de macro’s waar het in werd gezet kwamen er al snel meer veranderingen om de macro’s steeds te verbeteren.

Leerdoel 6: Onderzoeken – JavaScript

Elke dag werk ik met JavaScript. Met de opdracht: Strandeiland JS heb ik voor het eerst gewerkt met JavaScript op een andere manier die ik gewend ben. Daar was het echt doorbijten en goed begrijpen wat er staat. Ik zou op dit moment nog niet zelf de JavaScript kunnen schrijven die ik daar heb geschreven.

Om een Am chart te maken, doe je dat met JavaScript. Ik heb met de Am Chart opdracht het meest geleerd van JavaScript. In tegenstelling tot de Strandeiland JS opdracht, kan ik uit mezelf wel een Am Chart maken.

Leerdoel 7: Verbeelden en maken – HTML / CSS / JS

Met het maken van de Am charts, werd dat gedaan in macro’s waarbij Velocity werd gebruikt om HTML CSS en Javascript kan worden gebruikt op 1 plek. Niet iets wat ik zelf zou gaan doen later omdat ik op school heb geleerd om je bestanden te scheiden. Maar door Velocity kan dat.

Wanneer ik niet wist wat te doen, was ik begonnen met het maken van een Redesign van CrossmarX. Met die opdracht heb ik ook deze leerdoel kunnen voltooien. Daar is alleen CSS vervangen door Sass. Ik heb veel geleerd met het werken met Sass wat ik kan gebruiken met mijn carrière.

Observatieopdracht

Hierbij heb ik Rick Veltman geïnterviewd om meer te weten te komen over de 2 vraagstellingen. Al de 2 vraagstellingen worden beantwoord in het filmpje. Vraag 1 op 1.40 en vraag 2 op 6.21

Hoe wordt er samengewerkt binnen Crossmarx?

Kijk vanaf: 1.40

CrossmarX is een technisch bedrijf die heel erg goed is in ‘zelf dingen doen’. Iedereen zit graag zijn eigen ding te doen.

Samenwerken binnen CrossmarX gebeurt echt op ‘hoofdlijnen’. Er is een ‘ontwikkelagenda’ waar zaken in staan die zij belangrijk vinden. Een aantal van die zaken halen zij naar voren om er aan te kunnen werken.

CrossmarX is een software bedrijf die ‘marketing, verkoop’ heeft uitbesteed aan partners. Hun eigen marketing en verkoop aan partners, partijen die het leuk vinden om met hun tool te werken, daar moeten zij nog wel wat aan gaan doen. Samenwerking daarbij is nog wat zoekende. Wie gaat wat doen. Binnen de medewerkers van CrossmarX vindt niemand het echt leuk, maar dan is Rick toch wel weer degene die zich daar verantwoordelijk voor voelt. Maar om het in je eentje te doen is ook niet handig. CrossmarX gaat dit jaar stappen daarin maken om het te verbeteren. Het duurt nog wel eventjes. Het zal niet in een jaar klaar zijn.

Kern van samenwerken is dat wij met zijn alle wel een lange termijn idee hebben van waar het naar toe moet. Werknemers werken al wat langer bij CrossmarX waardoor zij zelf kunnen inschatten wat belangrijk is, daarin en waar zij zelf ook goed in zijn. Maar het is redelijk individueel. Rick zegt: Het is wel een beetje de aard van mensen in een technische achtergrond die graag staren naar een laptop.

In het verleden deed CrossmarX opdrachten zelf voor eindklanten. Nu hebben zij liever dat daar een partner tussen zit. Op dit moment is er een klant bij CrossmarX die al 15 jaar aan hun gebonden is door het maken van een applicatie. Die doen zij nog steeds rechtstreeks maar liefst wilt CrossmarX dat daar ook een partner tussen zit zodat die de eerste reeks vragen doen bij de klant en CrossmarX meer de moeilijkere dingen.

Die soort samenwerkingen dat CrossmarX intern klanten bedienen, die gaat wat minder worden. Klanten communiceren met CrossmarX via ‘issues’. Dan kijkt CrossmarX wie van de medewerkers het beste bij die vraag past om het op te lossen. Dat verdelen Roel en Rick. Zij kijken: Wat komt er binnen? Wie kan wat doen? En zo nu en dan moeten collega’s met elkaar ook overleggen. Dat kan op afstand of in een kantoor op het bedrijf. Die overleg momenten zijn vrij beperkt. Zo gaat dat binnen CrossmarX. Er zijn dus niet zoals grote project plannen of verantwoordelijkheden die belegd worden. Het wordt heel ligt georganiseerd.

Wat heb ik er van geleerd?

Door deze vraagstelling heb ik geleerd dat er op meerdere vlakgebieden er kan worden samengewerkt. Toen ik deze vraag stelde aan Rick stelde had ik in mijn hoofd: ‘samenwerken als collega’s binnen CrossmarX’. Had niet verwacht dat er ook nog iets bestond als: ‘samenwerken als partners binnen CrossmarX.

Door de vraag ben ik te weten gekomen hoe collega’s met elkaar samenwerken maar ook hoe CrossmarX samenwerkt met zijn klanten. Dit geeft mij een nieuw inzicht van hoe het ook moet. Door het weten van samenwerken binnen CrossmarX weet ik wat ik zo’n beetje wat ik later zou kunnen doen als ik een eigen bedrijf heb waar er samen moet worden gewerkt met werknemers. Ik zou het anders aanpakken dan hoe CrossmarX het doet. Ik zou liever hebben dat er samen wordt gewerkt i.p.v. zelfstandig samenwerken. Dat heb ik ook zo gezien op mijn oude stage bij Brick Zero. De manier hoe hun als team werken zou ik prettiger vinden dan hoe CrossmarX het doet. Maar zo leer je wel weer dat je je werknemers vertrouwen moet geven, wat CrossmarX goed doet.

Hoe worden keuzes vastgelegd waardoor het werk verder wordt geproduceerd?

Kijk vanaf: 6.21

Keuzes die schriftelijk wordt vastgelegd zijn er soms wel. Als er een applicatie ontwikkeld moet worden voor de klant gaan de medewerkers van CrossmarX voor een whiteboard staan met de klant waar keuzes worden gemaakt. Maar over het algemeen worden keuzes vastgelegd in issues. De vragen en de documentatie van het project zit in de issues die bij een applicatie horen. Issues kunnen opmerkingen zijn of een ontwerp. Het voordeel daarvan is dat elke medewerkers die bij het project zijn betrokken (ook niet CrossmarX medewerkers), toegang heeft om het te kunnen bekijken. Het is best anders dan bij een ander bedrijf. Bij CrossmarX zou het nooit echt een project management organisatie zijn. CrossmarX legt de project management ook bij de klant neer. Want zij willen wat. CrossmarX helpt en maken wat, maar de klanten willen wat. CrossmarX probeert dat ook zo ver mogelijk van hun af te houden. Anders vindt CrossmarX zelf ook dat zij daar anders een aparte dienst voor moeten regelen. Dat is niet wat hoort bij CrossmarX.

Wat heb ik er van geleerd?

Door deze vraagstelling heb ik geleerd dat CrossmarX grote keuzes schriftelijk wordt vastgesteld en dat kleine keuzes worden vastgesteld met issues. Voor die issues wordt de beste medewerker er aan vastgekoppeld om het op te lossen.

Reflectie

Terug kijken

Reflectie

Tijdens mijn stage heb ik goede kennis opgebouwd om verder te gaan als front-end developer. Op mijn stage heb ik geleerd dat je andere dingen moet leren om je doelen te kunnen behalen. Zo moest ik Velocity leren om verder te kunnen werken aan mijn JavaScript skills.

Waar ik het meest trots op ben, is mijn JavaScript kennis die ik heb opgedaan. Zowel als echt JavaScript als in combinatie met Velocity. In de eerste week van stage heb ik Velocity moeten leren. Ik kwam er snel achter dat ik het vond lijken op JavaScript waardoor ik er niet moeite mee had.

Waar ik het minst trots op ben is dat ik best wel veel tijd nodig had met het maken van de Am chart opdracht. Elke week kwamen er wel nieuwe dingen bij waardoor ik steeds nieuwe dingen moest toevoegen. Dit waren ook wel weer leermomenten voor mij geweest. Ik zie dat ik veel tijd nodig moet hebben om te leren ‘zelfstandig’ zijn. Zelfstandigheid is verbeterd maar het zou mooi geweest zijn als dat meer verbeterd zou zijn.

Tijdens mijn stage ging mijn Macbook kapot. Ik was mijn Macbook wel 5 weken kwijt waardoor ik moest werken op mijn oude Windows laptop. Gelukkig had ik die nog waardoor werk nog steeds geproduceerd kon worden.

In week 8 van mijn stage kreeg ik Corona waardoor ik 4 dagen niet kon werken. Maar gelukkig kan je dit vakgebied ook thuis uitvoeren.

Door de manier van werken binnen CrossmarX heb ik ervaringen kunnen opdoen om van te leren als ik zelf een bedrijf ga beginnen. Mijn JavaScript en Sass skills kan ik goed gebruiken in mijn verdere carrière als front-end developer.
